# Support

Thanks for using **InstantX**! This page lists the available channels for getting help, asking questions, and discussing the project.

## Documentation first

Before asking for help, please check the existing documentation:

- [README](./README.md) — project overview, architecture, and use cases.
- [Deployment Quick Start](./deployment/Quick-Start.md) — how to stand up the full stack locally.
- [Development guide](./docs/Development.md) — how to build, test, lint, and run static analysis.
- [User guides](./docs) — MQTT topic structure, V2X message standards, UPER encoding, Kafka, NiFi, and more.

## Asking questions and discussing the project

- **GitHub Discussions** — The primary place for questions, ideas, and general discussion: <https://github.com/lf-edge/instantx/discussions>. Discussions are searchable, addressable by URL, and open to everyone.
- **Stack Overflow** — Public Q&A tagged [`InstantX`](https://stackoverflow.com/questions/tagged/InstantX).
- **Project Wiki** — Community-maintained notes and how-tos: <https://github.com/lf-edge/instantx/wiki>.

## Reporting bugs and requesting features

- Open a [GitHub Issue](https://github.com/lf-edge/instantx/issues). Please read the [Contributing guide](./CONTRIBUTING.md) first for what to include in a good report.

## Reporting a security vulnerability

- **Do not** open a public issue or discussion for security problems. Follow the coordinated disclosure process in [SECURITY.md](./SECURITY.md).

## Maintainers

The people responsible for the project are listed in [MAINTAINERS.md](./MAINTAINERS.md).
